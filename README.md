# Xixi Voice Skill

A personal writing skill designed to preserve Xixi's voice:

- clear judgment
- strong structure
- human language
- grounded analysis
- emotional restraint without emotional coldness

This is not a generic "good writing" prompt.
It is a voice system for producing content that sounds like a specific person with a specific way of seeing reality.

## What This Skill Does

This repository helps turn raw topics, drafts, and ideas into content that sounds recognizably like Xixi.

It is built for tasks like:

- rewriting copy into Xixi's voice
- sharpening titles and openings
- diagnosing whether a topic fits the account
- turning flat questions into structural questions
- writing short video scripts, articles, and topic breakdowns
- preserving a consistent人格号表达方式

## Core Positioning

The shortest accurate定位 is:

**替普通年轻人解释现实，拆掉幻觉，留下方法。**

Expanded version:

**替年轻人拆现实、讲人性、给方法，但不贩卖焦虑。**

This voice is not trying to:

- perform intelligence
- vent emotion
- imitate social critics
- mass-produce content that only sounds "sharp"

This voice is trying to:

- explain what is really happening
- show the structure behind everyday困境
- keep empathy without surrendering judgment
- leave readers clearer, not just triggered

## Best-Fit Topics

This skill is strongest when used on topics like:

- young people's现实困境
- college, education, class, and effort
- family control, boundaries, and emotional roots
- work, money, risk, and decision traps
- human nature and growth
- grounded explanations of AI or new things for ordinary people

## How To Use

### In Codex or a compatible skill system

Place the folder where your skill runtime can load it, then trigger it with prompts such as:

- 用我的风格写
- 改成我的语气
- 按我的人格号来
- 帮我拆这个选题
- 把这个话题讲清楚但不要情绪输出

### As a writing reference

You can also use this repository manually:

1. Read `SKILL.md` for the core system.
2. Read `references/identity.md` to understand who Xixi is.
3. Read `references/writing-engine.md` before shaping a topic.
4. Read `references/language-patterns.md`, `references/sentence-bank.md`, `references/voice-corpus.md`, and `references/transformations.md` when writing.
5. Use `references/before-after-pack.md` when rewriting weak drafts.
6. Use `references/platform-playbooks.md` for format-specific output.
7. Use `references/editing-rubric.md` before finalizing.

## Writing Philosophy

This skill is based on a simple principle:

**不要把人骂醒。要把人讲明白。**

That means:

- do not answer the shallow version of a question
- rewrite the topic into a structure problem
- return to the ordinary person's real situation
- end with clarity, not empty comfort

## Quality Standard

If the output could have been written by any competent creator, it is not good enough.

The output should feel:

- recognizably personal
- structurally clear
- emotionally disciplined
- useful without sounding instructional
